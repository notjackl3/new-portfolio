March, 2025 - Resume Builder

Python, LaTeX, OpenAI, Jinja, Tkinter, PyQt5, OOP

I took what I've learnt in class about OOP to create my first personal project. This tool combines both CLI and Tkinter+PyQt5 for the interface, which takes in a resume and a job description, then uses textract to extract keywords. These keywords gets sent to OpenAI API to produce natural language sentences. These sentences are combined with Jinja and LaTeX to then output a newly formatted PDF resume. The new resume ensures keywords are incuded to improve ATS scores. The user may also change the document after if needed. A simple PostgreSQL database is used to store all the resumes and job descriptions locally.

April, 2025 - Youtube as a Service 👍

Python-Django, HTML/CSS/JS-Tailwind, PostgreSQL, Google Cloud Platform, AWS-S3, OpenCV, Numpy

I wanted to explore more. I noticed Youtube allows users to have unlimited video uploads, thus, I tried to find a loophole, which I found out that I can convert any files into binary, then match those binary values into specific pixel colors (e.g 00 is red, 01 is blue). These pixel colors are placed into a grid of 480x270 pixels in order. When the grid gets full, it continues onto a new one. Each of these grid images are then compiled into a video, each represents a frame. The video finally will be uploaded to Youtube. And users can upload as many video, files. I made a decompression algorithm such that I can redownload the video, tune resolutions, and retrieve the bits values. These are then recompiled back to the original file, with a 99.9% accuracy!

May, 2025 - Job Hunter 👍

Python-Django, HTML/CSS/JS, PostgreSQL-Supabase, AWS-EC2, NPTK, Jobspy, Beautiful Soup, OpenAI, Matplotlib, Chart.js

I wanted to avoid being flooded with job posts. This app helps you find the latest jobs by scraping Google, Linkedin, and Indeed based on input queries. From my previous keywords extraction knowledge in Resume Builder, this app collects and displays the most relevant, high demand technologies, along with other statistics, with addition of natural language processing algorithms. I also used Django's authentication and permission system for access control. There is an option for users to input their resume, and the app will produce a cover letter that matches different jobs immediately. All of the job postings are stored as a PostgreSQL database online via Supabase. The app is then deployed using EC2.
